# Aprendizajes de producto y mercado al 2026-03-13

## 1. Tecoloco: que si y que no

### Lo que si funciona de Tecoloco

- Va rapido al trabajo de buscar.
- El usuario entiende de inmediato donde escribir.
- Los filtros son familiares.
- La exploracion inicial no exige registro.
- El listado se puede escanear rapido.

### Lo que no se quiere copiar

- Home visual antigua.
- Densidad muy alta de bloques.
- Parte baja pesada.
- Estilo poco memorable.

### Conclusión

La referencia correcta no es:

- `copiar Tecoloco`

La referencia correcta es:

- `copiar su eficiencia de interacción`

## 2. Nicaragua: datos importantes para producto

Para filtros territoriales, la UI correcta es:

- `Departamento o region`

No solo `Departamento`, porque hay 15 departamentos y 2 regiones autonomas:

- Boaco
- Carazo
- Chinandega
- Chontales
- Esteli
- Granada
- Jinotega
- Leon
- Madriz
- Managua
- Masaya
- Matagalpa
- Nueva Segovia
- Rio San Juan
- Rivas
- RACCN
- RACCS

Esto debe mantenerse tanto en busqueda publica como en perfil candidato.

## 3. Claro Nicaragua: lectura de mercado usada

La lectura usada para orientar diseño y filtros fue esta:

- servicio al cliente / CAC
- ventas
- PYME
- distribucion
- campo
- telecom / fibra / soporte

No se debe diseñar el portal como si solo sirviera para puestos corporativos.

## 4. Señales que RH suele valorar

En este mercado, especialmente alrededor de perfiles similares a Claro, pesan mucho:

- comunicacion
- servicio
- metas
- Office / Excel
- movilidad
- licencia
- vehiculo
- disponibilidad para viajar
- horarios

Por eso se añadieron tags y campos alrededor de:

- licencia
- vehiculo
- viajes
- turnos
- Office
- servicio
- metas
- seguridad operativa
- nivel universitario

## 5. Regla de experiencia candidata

La mejor secuencia sigue siendo:

1. Buscar
2. Ver vacante
3. Registrarse si hace falta
4. Subir CV
5. Completar perfil
6. Postular

No al reves.

## 6. Regla de CV

La mejor solucion encontrada sigue siendo:

- `1 CV activo`
- historial automatico

No conviene mostrar muchas versiones para elegir en cada postulacion.

## 7. Regla de registro

El registro no debe pedir de inicio:

- foto
- direccion completa
- documentos pesados
- demasiados formularios

Debe pedir solo:

- nombres
- apellidos
- correo
- telefono opcional
- contrasena

## 8. Regla para RH

RH necesita eficiencia tambien.

Eso significa:

- filtrar desde lista
- no abrir detalle por detalle
- ver de un vistazo si el candidato tiene residencia alineada, categoria alineada y movilidad
